# BLACK RUSSIA Survival Game v3

Обновлённая версия сайта для Discord-ивента: браузерная игра на выживание с ведущим, invite-ссылкой, лобби, 20 более сложными раундами, жизнями, очками, таймером и спецраундами.

## Что изменено в v3

- Встроено 20 раундов вместо короткого набора вопросов.
- Вопросы стали сложнее: логика, ловушки, последовательности, вероятность, память, скорость и финальный риск.
- Улучшен дизайн игровых карточек, кнопок, панелей и состояния раунда.
- Исправлено лобби: больше нет наложения второго заголовка поверх картинки `ЛОББИ ИГРЫ`.
- Лобби теперь показывает аккуратную нижнюю панель со статусом игрока, жизнями, очками и количеством участников.
- В панели ведущего кнопка создания теперь создаёт игру с 20 раундами.

## Что внутри

- `index.html` — главная страница, игра и скрытая панель ведущего.
- `styles.css` — оформление в тёмном красно-чёрном стиле.
- `app.js` — логика игры и 20 встроенных раундов.
- `assets/hero.png` — баннер для главной страницы.
- `assets/lobby.png` — фон лобби игры.
- `assets/rules.png` — иллюстрация правил.
- `supabase/schema.sql` — SQL для базы Supabase.

## Как настроить Supabase

Если SQL уже выполнен и панель ведущего работает, заново выполнять схему не обязательно.

Для первой установки:

1. Открой проект Supabase.
2. Перейди в **SQL Editor**.
3. Открой файл `supabase/schema.sql`.
4. Скопируй весь код и нажми **Run**.

После этого сайт сможет создавать игры, подключать игроков и сохранять результаты.

## Как открыть панель ведущего

Панель скрыта с главной страницы. Открывается только по прямой ссылке:

```txt
/?host=1
```

Код ведущего:

```txt
JohnWick123
```

## Как применить новые 20 вопросов

Новые вопросы автоматически добавляются только в новые игры.

Если у тебя уже была создана старая игра, сделай так:

1. Открой панель ведущего.
2. Удали старую игру или создай новую.
3. Нажми **Создать игру с 20 раундами**.
4. Скопируй новую invite-ссылку игрокам.

## Как провести игру

1. Ведущий открывает `/?host=1`.
2. Вводит код ведущего.
3. Создаёт игру.
4. Копирует invite-ссылку.
5. Отправляет ссылку игрокам в Discord.
6. Игроки вводят ник и попадают в лобби.
7. Ведущий нажимает **Запустить игру**.
8. После каждого раунда ведущий нажимает **Следующий раунд**.
9. Игра заканчивается, когда остаётся один живой игрок или заканчиваются раунды.

## Правила игры

- У каждого игрока 3 жизни.
- За правильный ответ игрок получает очки.
- За ошибку или тайм-аут снимается жизнь.
- Если жизни закончились, игрок выбывает.
- В игре есть спецраунды:
  - `fast` — меньше времени на ответ;
  - `double` — правильный ответ даёт больше очков;
  - `immunity` — правильный ответ даёт щит;
  - `trap` — ошибка наказывается сильнее;
  - `reflex` — мини-игра на реакцию;
  - `memory` — вопрос на память;
  - `sudden` — внезапная смерть, ошибка почти гарантированно выбивает.
- Побеждает последний живой игрок. Если все раунды закончились, побеждает лидер по очкам.

## Как залить на GitHub Pages

1. Создай новый репозиторий.
2. Загрузи в корень репозитория:
   - `index.html`
   - `styles.css`
   - `app.js`
   - папку `assets/`
3. Папку `supabase/` после настройки базы лучше не заливать в публичный репозиторий, потому что в ней указан код ведущего.
4. В GitHub открой **Settings → Pages**.
5. Выбери **Deploy from branch** → `main` → `/root`.

## Важно про безопасность

Вход по коду `JohnWick123` удобен для закрытого Discord-ивента, но это не полноценная безопасная авторизация. Не публикуй ссылку `/?host=1` и код ведущего в открытом доступе.
